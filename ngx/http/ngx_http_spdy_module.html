Module ngx_http_spdy_module

Known Issues
Example Configuration
Directives
spdy_chunk_size
spdy_headers_comp
Embedded Variables

ngx_http_spdy_module模块为SPDY提供实验支持。 目前，已实施SPDY协议草案3.1 。

在1.5.10版之前，实施了SPDY协议草案2 。

默认情况下不构建此模块，应使用–with-http_spdy_module配置参数启用它。

该模块已被1.9.5中的ngx_http_v2_module模块取代。

已知的问题

该模块是实验性的，需要注意的是经纪人。

SPDY协议的当前实现不支持“服务器推送”。

在1.5.9之前的版本中，SPDY连接中的响应不能受速率限制 。

无论proxy_request_buffering ， fastcgi_request_buffering ， uwsgi_request_buffering和scgi_request_buffering指令值如何，都无法禁用客户端请求正文的缓冲。

示例配置 

server {
    listen 443 ssl spdy;


ssl_certificate server.crt;
ssl_certificate_key server.key;
...

}

请注意，为了在同一端口上同时接受HTTPS和SPDY连接，所使用的OpenSSL库应支持“Next Protocol Negotiation”TLS扩展，自Ope​​nSSL 1.0.1版开始可用。

指令


Syntax:

spdy_chunk_size size ;


Default:

spdy_chunk_size 8k;


Context:

http , server , location

该指令出现在1.5.9版本中。

设置响应主体被切片到的块的最大大小。 值太低会导致更高的开销。 由于HOL阻塞，太高的值会损害优先级。


Syntax:

spdy_headers_comp level ;


Default:

spdy_headers_comp 0;


Context:

http , server

设置响应的标头压缩level ，范围从1（最快，压缩程度较低）到9（最慢，最佳压缩）。 特殊值0关闭标头压缩。

嵌入式变量

ngx_http_spdy_module模块支持以下嵌入变量：

$spdy
SPDY连接的SPDY协议版本，否则为空字符串;
$spdy_request_priority
请求SPDY连接的优先级，否则请求空字符串。
